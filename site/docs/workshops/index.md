# Workshop Program — Planning · Building · Closeout

This program takes you from "I have an idea" to "I built it, proved it, and captured what I learned"
— using your own project, across three ordered parts.

Each part ships worksheets that work **self-paced** on this handbook: a reader who never attends a
live session can follow the same materials and reach the same artifacts. A live-facilitated cohort
uses the same worksheets and adds facilitation, cohort dynamics, and group presentations — not
different content.

---

## The three parts

Work through them in order. Each part hands off to the next.

### [Part 1 — Planning](./planning/index.md)

Take your project from a raw idea to an execution-ready plan: journal the raw idea, let Claude
interrogate you into a solid PRD, then decompose the PRD into an ordered list of individually-
verifiable tasks.

You leave Part 1 with a plan that can be handed to an autonomous run without further clarification.

### [Part 2 — Building](./building/index.md)

Run your plan through the three execution modes in order — monitored sequential, autonomous
subagent-driven, and multi-agent dynamic workflows — proving each result with a single deterministic
verification command rather than trusting the agent's self-report.

You leave Part 2 having run at least one task autonomously and proven it, handed off a multi-task
chunk and verified each result, and written a review standard for accepting autonomous output.

### [Part 3 — Closeout](./closeout/index.md)

Close the project you built with the same rigor you used to plan and execute it: a closing ceremony,
capturing learnings, carrying context forward into future work, and a brief presentation of your
results.

You leave Part 3 with a carry-forward artifact that seeds your next project, not a blank page.

---

## Your project through the program

You bring a project. Planning plans it. Building executes it. Closeout retrospects it.

This is a hard prerequisite: bring a low-stakes, real project you actually want to work on. A
partial idea is fine — Part 1 is designed to develop it into something executable through dialogue
with Claude. What you must not bring is someone else's gated or license-restricted material.

There is no prescribed sample project. The skills this program builds — spec-driven planning,
autonomous delegation with verification, deliberate close — transfer because you practice them on
real work, not a scripted capability checklist.

## Prerequisites

Before Part 1, confirm:

- **Your workspace is set up** — Claude Code installed, authenticated, and working.
  Follow [Getting Started](/docs/getting-started) if not.
- **You are comfortable with the lifecycle** — the spec-first, verify-don't-trust, and review
  discipline explained in [Working with Claude](/docs/working-with-claude).

Both pages are prerequisites, not suggestions. The program assumes the workspace and lifecycle
knowledge they cover.

## How the program delivers

**Live-facilitated cohort:** a facilitator leads each part as a structured session. The cohort
works together, presents to each other in Closeout, and surfaces patterns no individual would
have seen alone. The sessions are time-boxed; the pacing is set by the facilitator.

**Self-paced:** the same worksheets, completed at your own pace. Every worksheet reads coherently
without a facilitator present. Closeout's brief presentation becomes a written or self-recorded
share. Self-paced readers reach the same artifacts as cohort participants.

In both modes: the program's content is the worksheets. The live layer adds facilitation; it
does not introduce content that a self-paced reader lacks.
