# Planning Pre-Work

Complete this before the Planning session (or before you start the Planning worksheets
self-paced). The goal is to arrive with a working environment and at least a rough idea to work
with — so the session is about refining and planning, not setup and searching.

---

## Step 1 — Confirm your workspace is set up

You need Claude Code installed, authenticated, and connected to a repository you can experiment in.

If you haven't done this yet, follow [Getting Started](/docs/getting-started) first and return here
when the verification checks pass. Do not start the Planning session without a working environment.

**Check:**
- [ ] `claude --version` runs without error
- [ ] You are logged in (`claude` opens a session without prompting for a new login)
- [ ] Git is installed (`git --version` works)

---

## Step 2 — Bring a project idea

The Planning session is built around *your* project. You will use the worksheets to turn your idea
into a real executable plan — so the idea you bring becomes the thing you plan.

**What makes a good idea for this session:**

- Something you actually want to build or improve (even small)
- Low-stakes: you can experiment and iterate without risk to real users or important data
- Your own material: no license-restricted data, no credentials or data belonging to a third party
  you are not authorized to use, no sensitive personal information

You do not need a finished spec or a detailed vision. A rough idea is fine — the session is
designed to turn rough ideas into concrete plans. If you have several ideas, pick one to focus on;
you can run others through the worksheets as homework afterward.

**Bring only material you can freely share and experiment with.** If your idea involves a real
system or real data, create a sandbox version or a representative synthetic stand-in for use during
the session.

---

## Step 3 — Fill this before the session

Write a few sentences answering the prompts below. This is your raw-idea capture — you will refine
it during the session, so it does not need to be polished or complete. The point is that you arrive
with something written, not blank-page cold.

**Your raw idea (fill this in):**

> What is the thing you want to build or change? Write one to three sentences describing it as
> plainly as you can — what it does, who uses it, what problem it solves. Do not worry about
> being precise yet.
>
> *Example shape (not a template to copy): "A small tool that reads my project notes and surfaces
> the tasks I keep postponing. I want it to tell me what I am avoiding and why it might be blocked."*

## Ready check

Before you begin the Planning session or open the first worksheet:

- [ ] Workspace setup confirmed (Step 1 above)
- [ ] You have a project idea (Step 2 above)
- [ ] Your raw-idea prompts are filled (Step 3 above)

If you are joining a live cohort, bring this page with the prompts filled. If you are working
self-paced, keep it open alongside the
[PRD-Interrogation Worksheet](./prd-interrogation-worksheet.md) — that worksheet is your next step.
